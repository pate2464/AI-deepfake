4. BACKEND SETUP (Python / FastAPI)
================================================================================

Step 1: Open a terminal (PowerShell on Windows, Terminal on Mac/Linux)

Step 2: Navigate to the project root
    cd C:\Users\g18g1\AI-deepfake
    (or wherever you cloned/extracted the project)

Step 3: Create Python virtual environment
    python -m venv .venv

Step 4: Activate the virtual environment

    Windows (PowerShell):
        .venv\Scripts\Activate.ps1

    Windows (CMD):
        .venv\Scripts\activate.bat

    macOS / Linux:
        source .venv/bin/activate

    If you have an NVIDIA GPU:
        pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

    cd backend
    pip install -r requirements.txt

    This installs:
      - FastAPI, Uvicorn (web server)
      - Pillow, pillow-heif (image processing, HEIC support)
      - numpy, scipy, opencv-python-headless (signal processing)
      - google-generativeai (Gemini AI)
      - open-clip-torch, timm (CLIP model)
      - invisible-watermark (watermark decoder)
      - PyWavelets (wavelet transforms)
      - SQLAlchemy, aiosqlite (database)
      - And more...

Step 7: Verify installation
    python -c "import torch; print('PyTorch:', torch.__version__); print('CUDA:', torch.cuda.is_available()); print('GPU:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'None')"

    Expected output (with GPU):
      PyTorch: 2.5.1+cu121
      CUDA: True
      GPU: NVIDIA GeForce RTX 4070 Ti SUPER

Step 8: Set up environment variables (see Section 7 below)

The ML detection layers (CLIP, CNN) need pre-trained model weights.
A download script is provided.

Step 1: From the project root (with venv activated):
    cd C:\Users\g18g1\AI-deepfake

    If a download fails, follow the manual instructions printed.

  Windows: C:\Users\<you>\.cache\huggingface\

  Linux:   ~/.cache/huggingface/

    GEMINI_MODEL=gemini-2.5-flash

    # Debug mode (shows detailed logs)
    DEBUG=true

    # Database (auto-created)

return an error result with confidence 0.

      CLIP: HIGH (> 0.65)

  Upload an image for fraud detection.

  Content-Type: multipart/form-data
  Parameters:
    file (required)           — The image file to analyze
    account_id (optional)     — User account ID (for behavioral analysis)
    device_fingerprint (opt)  — Device fingerprint string

    delivery_lon (optional)   — Delivery longitude (GPS coordinate)
    order_value (optional)    — Order value in dollars
    claim_description (opt)   — Text description of the claim

  Returns: JSON AnalysisResponse with:
    - id, filename
    - risk_score (0.0 – 1.0)
    - risk_tier ("low", "medium", "high")
    - layer_results[] (13 items, each with layer, score, confidence, flags)
    - ela_heatmap_b64 (base64 PNG string)
    - trufor_heatmap_b64 (base64 PNG string, if available)
    - gemini_reasoning (text)
    - processing_time_ms
    - hash_matches[]
    - created_at

  Get past analysis results (newest first).

  Get detailed result for a specific analysis by ID.

--- GET /api/v1/stats ---
  Aggregate statistics: total scans, flagged count, avg score, layer triggers.

--- GET /api/v1/health ---

11. THE 13 DETECTION LAYERS EXPLAINED
================================================================================

Layer  Name              Weight   What It Does
─────  ────────────────  ──────   ────────────────────────────────────────
  1    EXIF Metadata     0.05     Checks camera make/model, GPS, timestamps.
                                  AI images have NO EXIF. Stripped EXIF = sus.

  2    ELA (Error Level) 0.08     Re-compresses image and compares error
                                  levels. AI images have eerily uniform ELA.

  3    Hash Matching     0.12     Perceptual hashing (aHash, pHash, dHash,
                                  wHash) to find duplicate/near-dup images
                                  in the database. Catches serial fraudsters.

  4    FFT Frequency     0.07     Fourier Transform of the image. GAN/diffusion
                                  models leave spectral artifacts invisible
                                  to the eye but visible in frequency domain.

  5    C2PA Provenance   0.03     Checks for Content Authenticity Initiative
                                  signatures. Adobe/camera C2PA proves origin.

  6    Behavioral        0.10     Analyzes user patterns: claim frequency,
                                  account age, device fingerprint, GPS vs
                                  claimed location. Catches repeat offenders.

  7    Gemini AI         0.07     Google Gemini 2.5 Flash analyzes the image
                                  for physical plausibility, contextual
                                  consistency, lighting, and manipulation.

  8    Noise / PRNU      0.05     Extracts camera sensor noise fingerprint
                                  using wavelet denoising. Real cameras have
                                  unique PRNU patterns; AI doesn't. Computes
                                  PCE (Peak-to-Correlation Energy) and
                                  spectrum flatness.

  9    CLIP AI Detect    0.18     CLIP ViT-L/14 extracts visual features,
                                  then a trained linear probe classifies
                                  real vs AI-generated. Highest-weighted
                                  layer. Based on UniversalFakeDetect paper.

 10    CNN Fingerprint   0.10     ResNet-50 fine-tuned on ProGAN with
                                  blur+JPEG augmentation. Generalizes well
                                  to other generators (~90% cross-gen accuracy).
                                  From Wang et al. CVPR 2020.

 11    Invisible WM      0.03     Decodes steganographic watermarks embedded
                                  by Stable Diffusion (DwtDctSvd 136-bit)
                                  and RivaGAN (32-bit). If found = AI image.

 12    TruFor Localize   0.07     Two-stream network (RGB + noise) for
                                  forgery detection & localization. Outputs
                                  a manipulation heatmap showing WHERE
                                  manipulation occurred.

 13    DIRE Reconstruct  0.05     Diffusion Reconstruction Error — feeds
                                  image through an autoencoder and measures
                                  reconstruction difference. AI images
                                  reconstruct differently. DISABLED by default.

  TOTAL WEIGHTS:         1.00

  Scoring: Weighted sum of (layer_score * layer_confidence * weight).
  Risk Tiers: LOW < 0.30 < MEDIUM < 0.60 < HIGH

12. GPU & PERFORMANCE NOTES
================================================================================

--- GPU Memory Usage ---
  After loading all models: ~6 GB VRAM
  During inference:         ~7-8 GB VRAM peak
  Your RTX 4070 Ti SUPER (16 GB) has plenty of headroom.

  Check live GPU usage:
    nvidia-smi

--- Performance Benchmarks ---
  First request (cold start — all models loading):
    ~2-3 minutes (includes CLIP backbone download first time)

  Subsequent requests (models cached in memory):
    ~10-15 seconds per image

  Breakdown by layer (approximate):
    Layers 1-8 (non-ML):   < 2 seconds combined
    Layer 9 (CLIP):         ~3-4 seconds
    Layer 10 (CNN):         ~1 second
    Layer 7 (Gemini API):   ~3-5 seconds (network latency)
    Layer 12 (TruFor):      ~2 seconds
    All layers run in parallel (asyncio.gather).

--- CPU-Only Mode ---
  If no NVIDIA GPU is detected, all ML layers automatically use CPU.
  Expect ~30-60 seconds per analysis instead of ~10s.

--- Reducing Memory Usage ---
  If you're low on VRAM:
    1. Models use float16 on GPU by default (torch.amp.autocast)
    2. Keep ENABLE_DIRE=false (saves ~200 MB VRAM)
    3. The system gracefully handles missing weights (falls back to heuristics)

13. TROUBLESHOOTING
================================================================================

PROBLEM: "ModuleNotFoundError: No module named 'app'"
  CAUSE:  Running uvicorn from wrong directory.
  FIX:    Use --app-dir flag:
          python -m uvicorn app.main:app --app-dir backend

PROBLEM: "CUDA not available" / torch.cuda.is_available() = False
  CAUSE:  Wrong PyTorch version installed (CPU-only).
  FIX:    Reinstall with CUDA support:
          pip uninstall torch torchvision
          pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

PROBLEM: "Port 8000 already in use"
  FIX:    Kill the existing process:
          Windows:  netstat -ano | findstr :8000
                    taskkill /PID <pid> /F
          Or use a different port:
          python -m uvicorn app.main:app --port 8001 --app-dir backend

PROBLEM: "Port 3000 already in use"
  INFO:   Next.js auto-picks the next available port (3001, 3002, etc.)
  NOTE:   Backend CORS allows all origins ("*"), so any port works.

PROBLEM: Frontend shows "Failed to fetch" or CORS error
  CAUSE:  Backend not running, or on a different port.
  FIX:    1. Ensure backend is running on port 8000
          2. Check http://localhost:8000/docs to confirm
          3. Frontend proxies /api/* to localhost:8000 via next.config.js

PROBLEM: "CNNDetection weights not found" warning in logs
  CAUSE:  Model weights not downloaded.
  FIX:    Run:  python download_models.py

PROBLEM: "image too small, should be larger than 256x256" (watermark layer)
  INFO:   This is normal for small images. The watermark decoder needs
          at least 256x256 pixels. Small test images will show
          "DwtDctSvd decode unavailable" — not a bug.

PROBLEM: Gemini layer returns error / score 0
  CAUSE:  Missing or invalid GEMINI_API_KEY in backend/.env
  FIX:    Get a key from https://aistudio.google.com/app/apikey

PROBLEM: First analysis takes forever (> 5 minutes)
  CAUSE:  CLIP ViT-L/14 backbone downloading (~900 MB, one-time).
  FIX:    Wait for it. Check network activity. Subsequent runs are fast.

PROBLEM: pip install fails on "invisible-watermark"
  FIX:    pip install invisible-watermark
          If that fails: pip install opencv-python-headless numpy
          Then retry.

PROBLEM: HEIC images not recognized
  CAUSE:  pillow-heif not installed or not registered.
  FIX:    pip install pillow-heif
          The app auto-registers it on startup (see main.py).

PROBLEM: "PermissionError" when writing to uploads/
  FIX:    Create the directory manually:
          mkdir backend\uploads

PROBLEM: Database is corrupted
  FIX:    Delete and let it recreate:
          python reset_db.py
          (Or just delete fraud_detector.db, it auto-recreates on startup)

14. RESETTING / CLEANING UP
================================================================================

--- Reset Database ---
    python reset_db.py
    # Or manually:
    del fraud_detector.db
    # It auto-recreates on next backend start.

--- Clear Uploaded Images ---
    Remove-Item backend\uploads\* -Force
    # Or on Linux: rm -rf backend/uploads/*

--- Clear Model Cache (re-download models) ---
    Remove-Item backend\models\clip_universalfakedetect\* -Force
    Remove-Item backend\models\cnndetection\* -Force
    python download_models.py

--- Clear CLIP Backbone Cache (huggingface) ---
    Windows:  Remove-Item $env:USERPROFILE\.cache\huggingface\hub\* -Recurse
    Linux:    rm -rf ~/.cache/huggingface/hub/

--- Full Clean Reinstall ---
    1. Delete .venv:           Remove-Item .venv -Recurse -Force
    2. Delete node_modules:    Remove-Item frontend\node_modules -Recurse -Force
    3. Delete database:        del fraud_detector.db
    4. Follow setup from Section 4 again.
